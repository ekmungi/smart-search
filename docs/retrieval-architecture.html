Scenario Cards

Click each card to expand details.

Find a file I vaguely remember PARTIALLY WORKS
"That Medtronic polyp-sizing paper"
Layers: L1 only
What happens: Filename + path fuzzy search -> file-level result
Current state: Semantic search finds chunks but does not aggregate to file-level results. User sees 10 chunks from 3 files instead of 3 files ranked by best match.
Browse documents by topic PARTIALLY WORKS
"User research files from last few months"
Layers: L1
What happens: Folder-aware search + date filtering -> file list
Current state: Keyword search works, but no date filtering and no file-level aggregation. Folder filter exists but no PARA awareness.
Deep-dive into a document MOSTLY WORKS
"Summarize the Methods section of that polyp study"
Layers: L1 -> L2 -> L4
What happens: Find file -> search within it -> LLM summarizes
Current state: read_note exists, but there is no within-document search to jump to a specific section. LLM must read full document and find the section itself.
Connect papers to meeting notes MOSTLY MISSING
"Were there meetings about this research topic?"
Layers: L1 -> L3 -> L2 -> L4
What happens: Find paper -> get related docs -> read meetings -> synthesize
Current state: find_related is on-the-fly only. No pre-built meeting-to-paper graph. LLM must manually orchestrate multiple search calls.
Multi-source synthesis PARTIALLY WORKS
"What do we know about X across papers, meetings, notes?"
Layers: L1 L2 L3 L4 All 4 layers
What happens: Discover files -> find connections -> read relevant -> synthesize
Current state: Requires manual LLM orchestration across multiple tool calls. No pre-built connections between documents. Works but is slow and may miss related documents.

User Journey Flow

Step-by-step walkthrough: "What do we know from user research?"

1
LAYER 1 File Discovery
knowledge_search(query="user research", result_type="files", limit=10)
Returns 10 documents: 3 PDFs, 2 DOCX, 5 MD notes
2
LAYER 2 Drill into promising docs
knowledge_search(query="user research findings", source_paths=[pdf1, pdf2, docx1])
Returns relevant chunks scoped to the 3 most promising documents
3
LAYER 3 Check for related meetings
knowledge_related(note_path=pdf1)
Returns: meeting_march.md (0.82), meeting_mike_april.md (0.76)
4
LAYER 2 Read the meeting notes
read_note("meeting_march.md"), read_note("meeting_mike_april.md")
Full text of both meeting documents loaded into context
5
LAYER 4 Synthesize
"You had 3 user research files. The March meeting discussed early findings. Mike's April meeting covered follow-up actions."

Quick Search vs Claude Code Boundary

Dimension Quick Search (UI) Claude Code (MCP)
Interaction Single query -> instant results Multi-turn conversation
Result type Files (titles, paths) Chunks, full docs, synthesized answers
Latency <200ms 1-10s acceptable
Intelligence Smart ranking, no LLM LLM-driven iterative search
Diversity HIGH many different files VARIABLE broad then deep
