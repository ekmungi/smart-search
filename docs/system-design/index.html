Personal Knowledge Base System

Inspired by Karpathy's LLM Wiki pattern, adapted for PARA-organized Obsidian vault, powered by Jeeves + Smart-search

The Big Picture: What This System Does

The Core Problem: You have hundreds of files across your Obsidian vault — PDFs, DOCX, PPTX, XLSX, meeting transcripts, research papers, product roadmaps, interview recordings. Each contains valuable knowledge, but it's locked inside opaque file formats. Finding connections across files requires remembering what's in each one. There's no way to ask "What do we know about FDA 510(k) pathways across all our meetings, research, and regulatory documents?" and get a synthesized answer.

The Solution: Build a system where every important file in your vault has an associated Markdown note (you already write many of these manually!). For files you haven't written notes for, an LLM automatically generates companion summaries. Then, across ALL these notes, the system extracts entities (products, people, regulations, companies) and concepts (regulatory strategies, clinical evidence frameworks) into cross-linked knowledge pages. The result is a living knowledge graph that grows smarter with every file you add.

Karpathy's Key Insight: Instead of using RAG (searching raw documents every time you ask a question), have an LLM do the hard work of synthesis, cross-referencing, and contradiction detection at ingest time. The wiki becomes a "persistent, compounding artifact" where the intellectual work has already been done. When you query it, the answers are fast and high-quality because the connections are pre-computed.

System Architecture: Four Layers

The system has four distinct layers, each with a clear responsibility. They work together but can evolve independently.

Layer 0 — Raw Sources

Your Files (Immutable)

These are the original files that live in your PARA-organized Obsidian vault. PDFs, DOCX, PPTX, XLSX, images, meeting transcripts, research papers. The system never modifies these. They are the source of truth. They live where they've always lived — inside your Projects, Areas, and Resources folders.

PDF DOCX PPTX XLSX PNG/JPEG MSG HTML CSV
↓
Layer 1 — Source Notes

Markdown Notes for Every File (User-authored or Auto-generated)

This is the bridge between opaque files and searchable knowledge. Every raw file should have an associated Markdown note that captures its key content in a text-searchable, LLM-readable format.

Your Existing Notes (Majority)

You already write meeting notes that link to transcripts, literature notes that reference PDFs, project notes that reference roadmaps. These are not duplicated. The system recognizes these as the companion note for the linked file and uses them as-is.

Auto-generated Companions (Gap-filling)

For raw files that don't have a note you wrote, the system auto-generates a companion MD in a _wiki/ subfolder. These are marked with auto_generated: true in their frontmatter so you always know what was human-written vs machine-generated.

↓
Layer 2 — Knowledge Graph

Cross-cutting Entity & Concept Pages (_knowledge/)

This is the compounding artifact — the layer that makes the system more than just a collection of notes. The LLM reads all your source notes (Layer 1) and extracts entities and concepts into dedicated pages that live in a vault-level _knowledge/ folder.

Entity Pages

Dedicated pages for specific things: products (CAMS AI Device), people (Dr. Smith), companies (Competitor X), regulations (FDA 510(k)), studies (CLEOPATRA-III Trial). Each page aggregates everything your vault knows about that entity, with source attribution.

Concept Pages

Pages for abstract topics: Regulatory Pathways for AI Medical Devices, Clinical Evidence Requirements, Market Dynamics in Digital Health. These synthesize knowledge across multiple sources into coherent reference articles.

Index & Log

index.md: Master catalog of all knowledge pages with one-line summaries. log.md: Chronological record of every ingestion event. Both are machine-maintained, human-readable.

↓
Layer 3 — Query & Search

Jeeves Agents + Search Engine

The interface layer where you (and Jeeves agents) ask questions and get answers. This layer searches across Layer 1 (source notes) and Layer 2 (knowledge graph) using semantic search (smart-search or QMD) and direct file reads. Jeeves agents like regulatory-sme, clinical-sme, and research-analyst become dramatically more effective because they can tap into pre-synthesized knowledge instead of raw documents.

Three-tier search:

  • Tier 0: Check _knowledge/ entity and concept pages first (fastest, highest quality — knowledge is already synthesized)
  • Tier 1: Semantic search via smart-search/QMD across all MD files (source notes + knowledge pages)
  • Tier 2: Grep/Glob fallback if search engine unavailable
